Tank game rules

Overview

Contest participants have to write a script that will fight in a multiplayer 2D platformer. The "game" consists of tanks placed on an 8x8 grid. It is turn based, and every turn, each tank can perform one of the following actions.
Move in any direction (up, right, down, left)
Shoot in any direction (up, right, down, left)
Moving means changing the current position of the tank (a tank can only move to a neighboring cell)
Shooting means covering a portion of a row/column with a deadly laser, that kills every tank it touches

How to play

Players create a Python script that will control their tank. They then upload their scripts to a server. At the end of the day, there is a showdown battle between all the players to see who programmed the best AI

Structure of the script:

All scripts should be written in python and contain the following function: 


            def play_turn(grid, player_pos): 

                do_something() 

                return action_code

Where grid is a 2D array of integers (the state of the board after the previous turn ended and:

  • grid[i][j] = 0 if the cell (i, j) is empty
  • grid[i][j] = 1 if the cell (i, j) has a tank
  • grid[i][j] = 2 if the cell (i,j) has a laser

And action_code is an integer representing the action to perform on the next turn. For example, 0 = Move up, 5 = Shoot right, etc

You can look at author's examples of basic scripts to get some inspiration (and use them as a template file): up_shooter.py random_mover.py persistent.py
